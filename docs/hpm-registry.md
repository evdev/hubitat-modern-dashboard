# Submitting to the HPM community registry

The package is installable today via a **custom repository URL** (see README). To appear in HPM search for all users, submit a pull request to the community registry.

## Steps

1. Fork [HubitatCommunity/hubitat-packagerepositories](https://github.com/HubitatCommunity/hubitat-packagerepositories).
2. Edit `repositories.json` and add an entry (adjust `name` if you prefer a display label):

```json
{
  "name": "evdev Modern Dashboard",
  "location": "https://raw.githubusercontent.com/evdev/hubitat-modern-dashboard/master/hubitat/repository.json"
}
```

3. Open a PR with a short description of the package.
4. After merge, users can enable your repository in HPM **Settings** without pasting a custom URL (it appears in the default repository list).

## Notes

- Keep `hubitat/repository.json` on `master` — the registry points at that file.
- Package version and release notes live in `hubitat/packageManifest.json` (generated by `npm run build`).
- Do not change package or app UUIDs in the manifest after users have installed.
